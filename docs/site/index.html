Load-bearing agent harness

Concern-aware evaluation for modern agent scaffolds.

LBAH wraps models, Fugu-style orchestrators, and external coding agents, then checks whether the right variables survived into the action that mattered. 

pip install -e ".[dev]"
lbah run --task moved_bottleneck:0 --agent configs/fugu_openai_compatible.yaml --mode audit --out runs/fugu_one

What makes it SOTA-shaped

Black-box harness support

Use OpenAI-compatible endpoints as agents while LBAH keeps the concern ledger and certificate loop.

Multi-agent handoff audits

Orchestration traces become transport and proxy evidence, so routing can be measured instead of trusted.

Evolution diagnostics

Run JSONL turns into failure families and concrete next experiments for harness improvement.

Run the loop

1

Install LBAH

Use editable install with dev dependencies so the CLI and tests share the local source tree.

2

Point at a harness

Set an API key and use one of the OpenAI-compatible configs or a local harness server.

3

Compare configurations

Run the same suite across agents and modes, then inspect success, load, cost, and failed gates.

4

Improve with evidence

Use the diagnostic report to choose the next transport, proxy, validator, or orchestration experiment.
